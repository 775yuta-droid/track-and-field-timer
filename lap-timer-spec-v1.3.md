# 1. 目的

トラック競技（800m / 1500m / 5000m）において  
**最小操作・高視認性・高信頼性でラップタイムを記録・共有する**

---

# 2. 前提条件

| 項目 | 内容 |
|------|------|
| 対象デバイス | iPhone（Safari / PWA） |
| 通信 | 初回のみ（以降オフライン） |
| 操作 | 片手（親指） |
| 使用環境 | 屋外（直射日光下） |
| 精度 | ±0.1〜0.3秒（人間操作含む） |
| 画面向き | 縦固定（portrait）。横回転は無効化する |

# 3. 対応種目

## 3.1 距離定義

```
events = {
  "800":  [400, 800],
  "1500": [300, 700, 1100, 1500],
  "5000": [200, 600, 1000, 1400, 1800, 2200, 2600, 3000, 3400, 3800, 4200, 4600, 5000]
}
```

> 5000mは `200 + 400×n (n=0〜12)` で計13ラップ。最終値: 200 + 400×12 = 5000 ✓　要素数: 13 ✓

# 4. 機能一覧

## 4.1 コア機能
- タイマー（実時間ベース）
- ラップ記録（累計＋区間）
- 距離ガイド表示
- 1ボタン操作

## 4.2 補助機能
- Undo（直前ラップ削除）
- 誤タップ防止（300msデバウンス）
- 固定スプリット欄（常時表示）
- ラップ行の奇偶グレー交互背景
- 種目プリセット切替
- 画像生成（共有用）
- RESET確認ダイアログ

## 4.3 廃止・非採用

| 機能 | 理由 |
|------|------|
| ラップ色分け（速い/遅い） | 種目依存で基準が曖昧、シンプルさを優先 |
| 自動スクロール | 新規ラップが上積みのため不要 |
| GPSラップ | 精度不足 |
| 音声操作 | 環境依存 |
| CSV共有 | 優先度低 |

# 5. 状態管理

```
idle → running → finished → idle
                 ↑
            Undo時: finished → running（startTime保持・タイマー継続）
```

## 5.1 状態遷移定義

| 遷移 | トリガー | 条件 |
|------|----------|------|
| idle → running | STARTタップ | state === "idle" |
| running → finished | 最終ラップ記録 | currentIndex === splits.length - 1 |
| finished → running | Undoタップ | state === "finished" かつ laps.length > 0 |
| finished → idle | RESET確認ダイアログで「リセット」 | state === "finished" |

> **Undo（finished時）**  
> `laps.pop()` + `currentIndex--` 実行後、stateを "running" に戻す。startTimeは保持したままタイマーを継続する。

# 6. 操作仕様

## 6.1 メインボタン

| 状態 | 表示 | 動作 |
|------|------|------|
| idle | START | 計測開始 |
| running（通常） | LAP | ラップ記録 |
| running（最終ラップ直前） | LAST LAP | 最終ラップ記録・finished遷移 |
| finished | RESET | 確認ダイアログを表示 |

> **「LAST LAP」表示条件**  
> `currentIndex === splits.length - 2` の時点でボタンラベルを `LAST LAP` に変更する。  
> 操作者が「次が最後」と事前に認識でき、誤操作・驚きを防ぐ。

## 6.2 RESET確認ダイアログ

- RESETタップ時、即座にリセットせず確認ダイアログを表示する
- ダイアログ内容：

```
「計測データを削除しますか？」
[ キャンセル ]  [ リセット ]
```

- 「リセット」タップ → finished → idle 遷移、全データ消去
- 「キャンセル」タップ → finished のまま何もしない

## 6.3 補助操作

| 操作 | 有効状態 | 動作 |
|------|----------|------|
| Undo | running / finished | 直前ラップ削除。finished時はrunningへ遷移 |
| 種目変更 | idle のみ | プリセット切替 |
| 画像保存 | finished のみ | PNG出力 |

## 6.4 Undoボタン配置

- 配置：画面**左上隅**（メインボタンと対角方向）
- サイズ：タップ領域 44px 以上
- 理由：親指によるメインボタン操作中に誤タップしにくい位置に固定

# 7. データ構造

```typescript
Lap = {
  index: number,        // ラップ番号（0始まり）
  timestamp: number,    // 記録時刻（Date.now()）
  cumulative: number,   // スタートからの経過時間（ms）
  split: number         // 直前ラップとの差分（ms）
}

AppState = {
  state: "idle" | "running" | "finished",
  startTime: number,    // Date.now() at START
  laps: Lap[],
  currentIndex: number, // 次に記録するラップのインデックス
  eventType: "800" | "1500" | "5000",
  splits: number[]      // 対応種目の距離配列
}
```

# 8. コアロジック

## 8.1 時間管理
- `Date.now()` を使用（OSシステム時刻ベース）
- 更新周期：50ms（`setInterval`）
- バックグラウンド移行・画面ロック時もシステム時刻は継続するため経過時間は正確に保持される

## 8.2 タイマー表示フォーマット

```
形式: mm:ss.t
例:   4:30.3

- mm : 分（0始まり、ゼロ埋めなし）
- ss : 秒（2桁、ゼロ埋めあり）
- t  : 1/10秒（1桁）
```

> 精度仕様が±0.1〜0.3秒のため、1/100秒（2桁）表示は過剰で誤解を招く。1/10秒が精度と表示の整合性として適切。

## 8.3 ラップ計算

```
cumulative = Date.now() - startTime
split = cumulative - laps[laps.length - 1].cumulative  // 初回は cumulative そのまま
```

## 8.4 Undo

```
laps.pop()
currentIndex--
if (state === "finished") state = "running"
// startTime はそのまま保持
```

> 下限: `laps.length === 0` の場合は処理無視（§14参照）

## 8.5 誤タップ防止
- 直前の有効タップから300ms以内の入力を無効化（デバウンス）

## 9.3 ラップ一覧（v1.3改定）

### 並び順・スクロール

| 項目 | 仕様 |
|------|------|
| 並び順 | **新しいラップが上**（上積み方式） |
| 自動スクロール | **廃止**（新規が常に最上部のため不要） |
| スクロール | 手動。下に向かって過去ラップを参照可能 |

### 各行フォーマット

```
Lap N   [距離]m   [区間タイム]   [累計タイム]
```

**表示例（1500m）：**
```
Lap 3  1100m  1:13.5  3:34.0
Lap 2   700m  1:12.3  2:20.5
Lap 1   300m  1:08.2  1:08.2
```

### 各要素のスタイル

| 要素 | フォントサイズ | 色 | 備考 |
|------|---------------|----|------|
| `Lap N` | 12px | #666666 | 補助情報 |
| 距離 | 12px | #666666 | 補助情報 |
| 区間タイム | 18px | #FFFFFF | 主役・monospace |
| 累計タイム | 18px | #AAAAAA | 副役・monospace |

> 区間と累計を同サイズにしつつ、累計を `#AAAAAA` に落とすことで視覚的な階層を持たせる。固定スプリット欄（48px）が区間タイムの「今」を担うため、一覧内での区間・累計は対等に扱う。

### ラップ行背景（奇偶交互）

| ラップ行 | 背景色 |
|----------|--------|
| 奇数行（1, 3, 5…） | #1A1A1A |
| 偶数行（2, 4, 6…） | #000000 |

```css
.lap-row:nth-child(odd)  { background-color: #1A1A1A; }
.lap-row:nth-child(even) { background-color: #000000; }
```

> 並び順が新規上積みのため、画面上部の奇数・偶数は記録が増えるたびに入れ替わる。視覚的なシマ模様の目的は「行の区切り識別」であり、特定ラップへの固定色付けは不要。


# 13. エラー処理

| ケース | 対応 |
|--------|------|
| 300ms以内の連打 | 入力無効（デバウンス） |
| `laps.length === 0` でUndo | 処理無視 |

# 14. UX設計思想

- **1ボタン原則**：メイン操作はSTART / LAP / LAST LAP / RESETの単一ボタンに集約
- **視認性最優先**：黒背景・白文字・大型フォントで直射日光下でも判読可能
- **情報の役割分担**：固定スプリット欄（今）とラップ一覧（過去）で明確に分離
- **ミス前提設計**：Undo + デバウンス + RESET確認の三重防衛
- **事前通知**：LAST LAPラベルで最終ラップを予告し、操作者の準備を促す
- **即共有**：計測終了後すぐにSNS投稿可能な画像を生成

# 15. リスク対策

| リスク | 対策 |
|--------|------|
| 誤タップ（LAP） | 300msデバウンス + Undo |
| 誤タップ（RESET） | 確認ダイアログによる二段階確認 |
| Undoの誤タップ | 左上隅配置（メインボタンと対角） |
| 直射日光下の視認性低下 | #000000背景 + #FFFFFF文字（コントラスト比 21:1） |
| ラップ行の見間違い | 奇偶グレー交互背景 |
| 最終ラップの見落とし | LAST LAPラベルで事前通知 |
| 操作状態の混乱 | 状態機械による厳密な遷移制御 |
| バックグラウンド移行による時刻ズレ | Date.now()使用によりOS時刻ベースで正確 |
| 画面の誤回転 | portrait固定（PWAマニフェスト + CSS） |
| Canvas描画のフォント差異 | フォントファミリーを明示指定 |
| スクロール迷子 | 新規ラップが常に最上部のため現在位置が自明 |
